COBOL_OS, My Ambitious Vaporware

So... You're probably here because you're either:

Bored as all hell

Tired of normal OS

Found me on the web

Or wondering, "How the hell does one code an OS in COBOL?"

Or all four, even! Four is good! Either way, it is important for any viewer to know:

This project, although bizarre, is a genuine piece of software I am trying to work on. I am not programming a calculator in Malbolge. I am dead-seriously, unironically, programming an alternative OS for x86 (and maybe even ARM64) in COBOL. The goals are simple: An init, memory management, display, keyboard control, CRUD and a rudimentary shell.

Currently, this project is still in its infancy. I am now actively working on it, and anyone who wants to contribute is heavily encouraged to submit pulls to ~~fix my big ball of mud~~ help me out.
